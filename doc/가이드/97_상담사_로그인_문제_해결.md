# 상담사 로그인 가이드

## 자동 생성 (권장)

애플리케이션 시작 시 **DataLoader**가 자동으로 상담사 계정을 생성합니다.

### 로그인 정보
- **학번**: 8000001 또는 8000002
- **비밀번호**: counselor123

### 확인 방법
1. 애플리케이션을 시작합니다
2. 로그에서 다음 메시지를 확인합니다:
   ```
   상담사 계정 생성: 김상담 (학번: 8000001, 비밀번호: counselor123)
   상담사 계정 생성: 이상담 (학번: 8000002, 비밀번호: counselor123)
   ```

## 문제 해결

### 1. 계정이 이미 존재하는 경우

데이터베이스에 상담사 계정이 이미 있지만 로그인이 안 되는 경우:

```sql
-- MySQL 접속
mysql -u root -p scms2

-- 상담사 계정 확인
SELECT user_id, student_num, name, email, role FROM users WHERE role = 'COUNSELOR';

-- 비밀번호가 잘못된 경우, 상담사 계정 삭제 후 재시작
DELETE FROM counselors WHERE user_id IN (SELECT user_id FROM users WHERE student_num IN (8000001, 8000002));
DELETE FROM users WHERE student_num IN (8000001, 8000002);
```

애플리케이션을 재시작하면 DataLoader가 올바른 계정을 다시 생성합니다.

### 2. 수동으로 추가하기

DataLoader가 작동하지 않는 경우:

```bash
# add-counselor.sql 실행
mysql -u root -p scms2 < docs/add-counselor.sql
```

### 3. 로그 확인

로그인이 여전히 안 되는 경우, 애플리케이션 로그를 확인합니다:

```bash
# 콘솔 로그에서 "상담사 계정 생성" 메시지 확인
# 또는 로그에서 에러 확인
tail -f logs/application.log | grep -i "counselor\|상담사"
```

## 상담사 계정 정보

| 학번 | 이름 | 이메일 | 비밀번호 | 전문분야 |
|------|------|--------|----------|----------|
| 8000001 | 김상담 | counselor1@pureum.ac.kr | counselor123 | 진로상담, 학업상담 |
| 8000002 | 이상담 | counselor2@pureum.ac.kr | counselor123 | 심리상담, 대인관계 |
